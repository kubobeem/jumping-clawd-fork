# readmeja.md 仕様書

> 作成日: 2026-06-22
> リクエスト元: ユーザー
> 対象プロジェクト: jumping-clawd-fork

## 1. 概要

### 1.1 目的
Jumping Clawd フォークの**日本語README**（`readmeja.md`）を作成する。これは既存の英語 `README.md` の単なる翻訳ではなく、日本語話者向けに最適化された独自のREADMEとする。

### 1.2 背景
- プロジェクトはブラウザ拡張機能のミニゲーム「Jumping Clawd」
- 既に `public/_locales/ja/messages.json` でUIの日本語対応が完了済み
- 英語の `README.md` は存在するが、日本語のREADMEは未整備
- フォークプロジェクトであり、元との違いを説明する価値がある

### 1.3 位置づけ
- ファイル名: `readmeja.md`
- プロジェクトルート（`README.md` と同じ階層）に配置
- `README.md` を置き換えるものではなく、**補完的な日本語版**として共存

## 2. トーンとスタイル

| 項目 | 内容 |
|------|------|
| **トーン** | バランス型 — ユーザー向け情報と開発者向け情報の両方をカバー |
| **言語** | 日本語（標準的・丁寧な文体） |
| **フォーマット** | 標準Markdown（絵文字なし） |
| **ボリューム** | 既存 `README.md` と同程度 |
| **対象読者** | 日本語話者のエンドユーザー＋開発者 |

## 3. 含めるセクション（優先度順）

全て「必須」として実装する。

### 3.1 遊び方・操作方法（必須）
- ゲームの目的と基本ルールの説明
- 操作方法:
  - **カジュアルモード**: Ctrl + `,` で開始、Space長押しでチャージ、離してジャンプ
  - **チャレンジモード**: Ctrl + `.` で開始、時間経過で床が上昇する
  - **Auto Play**: Ctrl + `A` で自動プレイのON/OFF切り替え
  - **終了**: `Esc` キーでオーバーレイ終了
  - オーバーレイモードとスタンドアロンタブモードの違い
- スコアとランキングシステムの説明
- トップスパイク（天井）とボトムスパイク（床）による死亡条件

### 3.2 インストール方法（必須）
- 前提条件: Node.js >=20.12.0
- `npm install` による依存関係のインストール
- Chrome拡張機能としての読み込み手順（デベロッパーモード）
- Firefoxでの利用方法

### 3.3 プロジェクト構成（必須）
- ディレクトリ構造の概要
- 主要ファイルの役割説明:
  - `entrypoints/` — 拡張機能のエントリーポイント（バックグラウンド、ポップアップ、ゲームページ、オーバーレイ）
  - `src/game/` — ゲームロジック全般
  - `src/extension/` — 拡張機能側の共通ロジック
  - `public/_locales/` — 多言語対応ファイル
  - `wxt.config.ts` — WXT（ブラウザ拡張ビルドツール）の設定

### 3.4 開発環境セットアップ（必須）
- `npm run dev` — Chrome開発モード起動
- `npm run dev:firefox` — Firefox開発モード起動
- `npm run compile` — 型チェック
- `npm run build` / `npm run zip` — ビルド・パッケージング
- `AGENTS.md` の存在と開発の注意点

### 3.5 カスタマイズ方法（必須）
- ゲームパラメータの調整: `src/game/config.js`
  - プラットフォーム間の距離、ジャンプタイミング、キャラクターサイズ
  - スパイクの高さ、チャージメーターの設定
  - チャレンジモードの難易度パラメータ
- アニメーションの調整: `src/game/clawd-motion.js`
- スタイルの変更: `src/game/styles.css`
- リーダーボードの設定変更:
  - `src/game/leaderboard.js` の Supabase 設定
  - `wxt.config.ts` の `host_permissions`
- 新言語の追加方法: `public/_locales/` に新しいディレクトリを作成

### 3.6 Git管理について
- コミットすべきファイルと `.gitignore` で除外されるファイルの説明
- 生成ファイル（`.wxt/`, `.output/`, `node_modules/`）の扱い

### 3.7 フォークについて（補足）
- フォーク元との違いを簡潔に記載
- リリースノート（`release-notes.md`）への言及

## 4. 既存 README.md との差異

| 項目 | README.md（英語） | readmeja.md（日本語） |
|------|-------------------|----------------------|
| **言語** | 英語 | 日本語 |

| **絵文字** | あり | なし |
| **遊び方** | 簡潔 | やや詳しく（UI翻訳キーの対応説明を含む） |
| **リリースノート参照** | なし | あり（フォークの経緯に言及） |

## 5. 非機能要件

- `README.md` と同じMarkdownの品質基準を満たす
- コードブロック内のコマンド例は英語のまま維持（翻訳しない）
- ファイルパスや設定キー名は原文のまま表記
- `chrome.i18n.getMessage()` のキー名など、コード要素は翻訳しない
- リンク先（URL）は翻訳しない

## 6. 実装メモ

- ファイル作成のみ（既存ファイルへの変更はしない）
- ゲームコードや設定への変更は一切行わない
- `README.md` から `readmeja.md` への相互リンクは現時点では不要
- 作成後、Git管理対象としてコミット候補に含める
