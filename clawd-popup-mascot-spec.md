# Clawd ポップアップマスコット 仕様書

> 作成日: 2026-06-22
> プロジェクト: jumping-clawd-fork
> 機能名: Clawd Browser Mascot（Clawd ブラウザマスコット）

## 1. 概要

### 1.1 コンセプト
現在のJumping Clawdは「ゲーム内でプラットフォーム間をジャンプするだけ」。これを拡張し、Clawdキャラクターが**ブラウザの中を自由に動き回るポップアップマスコット**として動作する機能を追加する。いわば「You are an idiot」ポップアップの進化系ミニキャラとして、Clawdがブラウザ画面上をうろちょろする。

### 1.2 機能の位置づけ
- **既存のゲームとは独立した機能**
- 拡張機能ポップアップからモード選択して起動/停止できる
- ゲームと同時に動かすことも可能（排他的ではない）

## 2. 動作モード（3モード）

### 2.1 Bモード: Webページ上ホッピング
Clawdが現在開いているWebページの**要素（テキストの単語、画像、リンク、ボタンなど）を足場**として、その上をピョンピョン跳ね回る。

- **ページ要素とのインタラクション**: なし（純粋に見た目のアニメーションのみ。リンクを踏んでもクリックは発生しない）
- **足場の検出**: ページ上のDOM要素を検出し、その位置をプラットフォームとして利用
- **移動パターン**: 要素間をジャンプして移動。一定時間そのエリアにとどまったら次のエリアへ
- **制限**: スクロールに合わせてClawdも追従する（ビューポート内に留まる）

### 2.2 Cモード: ブラウザUIマスコット
ClawdがブラウザのUI部分（ツールバー、タブバー、アドレスバー周辺、ウィンドウのフチなど）を跳ね回るデスクトップマスコット。

- **行動範囲**:
  - ツールバーエリア（拡張機能アイコンやURLバーの周辺）
  - ブラウザのフチ（ウィンドウの端っこをぐるぐる）
  - タブバー（開いているタブの上を歩く）
  - 画面の四隅（画面端で跳ね返るボールのようにバウンド）
- **すべてのエリアを行き来できる**

### 2.3 ポップアップマスコットモード（メイン機能）
「You are an idiot」のポップアップのように、**小さなClawdウィンドウ**がブラウザ画面上に常駐し、自由に動き回る。BモードとCモードはこの内部の挙動パターンとして切り替える。

#### 2.3.1 動作パターン（全て実装、状況に応じて変化）

| パターン | 説明 | トリガー |
|---------|------|---------|
| **カーソル追従** | マウスカーソルの近くをうろちょろする | マウスがアクティブに動いているとき |
| **ランダム移動** | 画面内を自由気ままに跳ね回る | アイドル時、カーソルが長時間動いていないとき |
| **画面端バウンド** | 画面端にぶつかると跳ね返るアニメーション | 端に到達したとき |
| **要素に興味示す** | ボタンやリンクの近くで止まって覗き込む | インタラクティブ要素の近くを通るとき |
| **クリックでジャンプ** | クリックすると驚いてジャンプする | Clawd本体をクリックしたとき |

#### 2.3.2 キャラクター状態アニメーション

| 状態 | 説明 | 発生条件 |
|------|------|---------|
| **歩く** | 左右にゆっくり歩くアニメーション | 移動中（通常時） |
| **跳ねる** | ピョンピョン跳ねながら移動 | 機嫌が良いとき／餌をあげた直後 |
| **座る／待機** | その場に座って休憩 | 長時間操作がないとき |
| **寝る** | 目を閉じてZzzを出す | 非常に長い間放置されたとき |
| **こっちを見る** | マウスが近づくと首を向ける／目で追う | マウスホバー時 |

#### 2.3.3 インタラクション機能

| 操作 | 挙動 |
|------|------|
| **クリック＆ホールドでなでる** | マウスを押したまま動かすと撫でられる感覚。Clawdが気持ちよさそうな表情になる |
| **ホバーで反応** | マウスが近づくとClawdがこちらを向き、目をまあるくする／瞬きする |
| **ドラッグで持ち上げ移動** | Clawdをつかんで画面上の別の場所にドラッグ＆ドロップ |
| **表情変化（なでなで）** | 撫でると目がハートになる／とろけた表情になる |
| **餌をあげる** | ダブルクリック（または特定操作）で餌をあげるとClawdが食べるアニメーション。満足度ゲージが上がる |

## 3. UI・設定（拡張機能ポップアップ）

既存のポップアップUIに以下の設定項目を追加する。拡張機能アイコンをクリックして開くポップアップ内で全て設定可能とする。

### 3.1 モード選択

```
┌──────────────────────┐
│  🦀 Clawd Mascot     │
├──────────────────────┤
│  ○ オフ（非表示）     │
│  ○ Webページ上を跳ねる │
│  ○ ブラウザUIを跳ねる  │
│  ○ ランダム（自動切替）│
├──────────────────────┤
│  [詳細設定 ▸]          │
└──────────────────────┘
```

### 3.2 詳細設定パネル

| 設定項目 | 種類 | デフォルト | 説明 |
|---------|------|-----------|------|
| **サイズ** | スライダー | 48px | Clawdの表示サイズ（16px〜128px） |
| **透明度** | スライダー | 100% | 背景透明度（0%〜100%） |
| **お仕事モード** | トグル | OFF | ONにすると動きが最小限になり、音も出ない |
| **節電スリープ** | トグル | ON | アイドル時にスリープしてCPU節約 |
| **クリック透過** | トグル | OFF | ONにするとClawdをクリックしてもWebページが操作できる |
| **餌の残量** | ゲージ表示 | - | 現在の満腹度（餌をあげると増える） |

### 3.3 表示条件
- 全てのWebページで表示可能（許可リスト制ではない）
- 現在のゲーム（オーバーレイモード）と同時表示可能

## 4. 技術的設計方針

### 4.1 ブラウザ対応
- **Chrome / Chromium系（Edge, Brave, Vivaldi含む）** — フルサポート
- **Firefox** — 同等の機能を提供（WXTのビルドターゲットで対応）
- 拡張機能APIの差異はWXTフレームワークで吸収

### 4.2 実装方式
- **浮遊ポップアップ**: 拡張機能のオーバーレイ機構（既存の `page-game-overlay.ts` と類似）を再利用
- **DOM操作**: Shadow DOMを使用してページのCSS影響を受けないようにする
- **アニメーション**: CSSアニメーションとCanvas（またはSVG）の組み合わせ
- **イベント**: `mousemove`, `click`, `scroll`, `resize` を監視

### 4.3 パフォーマンス
- アイドル検出: マウス操作がないと自動的にスリープ状態へ移行
- フレームレート制御: アクティブ時は30fps、アイドル時は1fpsに低下
- `requestAnimationFrame` + `IntersectionObserver` で描画最適化
- ポップアップ非表示時はリソースを完全に解放

### 4.4 競合対策
- Vimium, crxMouse など競合する拡張機能が存在する場合でも動作
- `pointer-events`, `z-index` の適切な設定で他のUIとの干渉を最小化

## 5. ファイル構造（案）

```
src/mascot/
├── index.ts              # エントリーポイント（content script）
├── mascot-state.ts       # 状態管理（モード、位置、アクション）
├── mascot-renderer.ts    # 描画エンジン（SVG/Canvas）
├── mascot-behaviors.ts   # 行動パターン（追従、ランダム、バウンド等）
├── mascot-pet.ts         # なでなで・餌やりインタラクション
├── mascot-animations.ts  # アニメーション定義
├── mascot-styles.css     # Shadow DOM用スタイル
└── types.ts              # 型定義

entrypoints/popup/
├── main.ts               # 既存ファイルにマスコットUIを追加
└── style.css             # 既存ファイルにマスコット設定UIスタイル追加

entrypoints/mascot-overlay.ts  # 新規: マスコット用content script
```

## 6. 実装の優先順位（推奨）

| フェーズ | 内容 | 理由 |
|---------|------|------|
| **Phase 1** | 浮遊ポップアップの基本表示＋ランダム移動＋カーソル追従 | コア機能の確立 |
| **Phase 2** | Webページ上の要素検出＋要素間ホッピング | Bモードの実現 |
| **Phase 3** | ブラウザUI領域の検出＋端バウンド | Cモードの実現 |
| **Phase 4** | なでなで・ドラッグ・表情変化 | インタラクション追加 |
| **Phase 5** | 餌やり・状態遷移（座る・寝る等） | キャラクター深化 |
| **Phase 6** | 設定UI（ポップアップ）・サイズ調整・節電 | ユーザーコントロール |
| **Phase 7** | Firefox対応・パフォーマンスチューニング | 安定化 |
